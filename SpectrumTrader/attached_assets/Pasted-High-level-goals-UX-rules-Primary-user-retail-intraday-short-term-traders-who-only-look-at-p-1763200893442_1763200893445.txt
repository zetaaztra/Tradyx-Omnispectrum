High-level goals & UX rules

Primary user: retail intraday & short-term traders who only look at points and a one-line verdict. UI must surface numeric point forecasts prominently, then supportive visuals (range bands, mini-charts) secondarily.

Tone: institutional, authoritative, minimal noise — users must feel “secure” (calm UI, clear numbers).

Must avoid probabilistic jargon on primary elements — show point values and simple labels (UP / DOWN / FLAT / SELLERS: OK / BUYERS: CAUTION). Under-the-hood JSON stores probabilities; expose only via a secondary detail panel or tooltip.

Accessibility: sufficient contrast in both modes, keyboard focusability, readable sizes.

Performance: static JSON fetch on load and periodic refresh (30m) — animations are subtle (no heavy DOM work). Use next/image sparingly; charts rendered in canvas or SVG (Recharts / Chart.js).

2 — Tech stack (recommended)

Framework: Next.js 14 (App Router) — server-side static site generation for fast loads; Cloudflare Pages can serve static built output.

Styling: Tailwind CSS (utility-first, consistent tokens).

Charts & gauges: Recharts (React, SVG) for mini charts and area bands; React-SVG-Gauge or custom SVG gauge for numeric dial; for advanced interactive charts Chart.js (react-chartjs-2) or Plotly optional but heavier.

Animations: Framer Motion for subtle transitions (tile entrance, gauge sweep).

Icons: Lucide / Heroicons (open-source).

Fonts: see Typography below (Inter + Space Mono).

State & data: simple React hooks + SWR for data fetch/cache and revalidation.

Build & CI: GitHub Actions for inference (writes JSON), Cloudflare Pages for hosting.

3 — Typography (recommended free fonts)

UI font (primary): Inter — modern, neutral, variable/weights. Use for most UI text.

sizes: Body = 14px, small = 12px, large labels = 16–18px, tile titles = 14–16px.

Numeric / Monospaced Display: Space Mono or IBM Plex Mono for the big point numbers to emphasize numeric clarity. Use a large weight for the main forecast.

Font weights & scale (desktop baseline):

h1 / large numbers: font-family: "Space Mono", Inter; font-weight: 700; font-size: 48px

tile title: Inter, 14px, 600

body: Inter, 14px, 400

caption: Inter, 12px, 400

Accessibility: min text contrast ratio WCAG AA. Use semi-bold for short labels.

4 — Color system & tokens

Design principle: calm, high-contrast, and clear semantic colors for market direction.

Dark mode (primary preferred)

Background: --bg: #0B0F14 (near-black, not pure)

Surface (cards): --card: #0E1620 (soft navy)

Primary accent (Bull/positive): --bull: #67E480 (greenish)

Bear/negative: --bear: #FF6B6B (muted red)

Text primary: --text: #E6EEF3

Muted text: --muted: #9FB0BD

Accent for UI controls: --accent: #6BC7FF (cyan)

Light mode (secondary)

Background: --bg: #F6F9FB

Surface: --card: #FFFFFF

Text primary: #0B1720

Bull: #017A3A (deeper green)

5 — Layout & grid

Overall grid (desktop)

Page container width: max-width: 1280px centered.

Primary grid: CSS grid grid-template-columns: 1fr 420px on wide screens. Left: main tiles; Right: summary / action panel. On narrow screens collapse to single-column.

Tiles grid (main)

Use 3-column grid for main tiles (desktop): grid-template-columns: repeat(3, 1fr); gap: 16px.

Tiles sizes:

Main FORECAST tile (Tomorrow point) — spans 2 columns × 2 rows (large prominent card top-left).

Secondary numeric tiles (2-day, 3-day) — smaller cards, single cell adjacent.

Range bands and envelopes are medium cards; PatternMatch and Signal tiles small cards.

Mobile

Single column, stacked ordering: Tomorrow → 2d → 3d → week → month → envelopes → option radars → pattern → composite sentence.

8 — Dark vs Light mode specifics

Dark mode (recommended default)

Card background: --card navy (#0E1620) with 1px soft border rgba(255,255,255,0.03).

Primary numbers: --text white; positive numbers --bull highlighted with soft glow (box-shadow: 0 6px 20px rgba(103,228,128,0.12)) to create calm reassurance.

Range bands: semi-transparent fills; bullish fill rgba(103,228,128,0.12), bearish rgba(255,107,107,0.10), neutral rgba(155,165,173,0.06).

Sparkline strokes: --accent (cyan) for upward trending; --neutral for flat.

Gauge/Traffic light: subtle neon border.

Micro-animations: gauge sweep on value change (0.5s ease-out), tile load fade + upward translate (framer-motion).

Light mode

Card background: white #ffffff, shadow 0 6px 18px rgba(14,22,32,0.06).

Primary numbers: #0B1720 text; bull color #017A3A, bear #B00020.

Range fills: very light greens/red tints.

Keep same iconography & layout, invert glow to drop-shadow.

Mini example JSX for Tomorrow tile:

export default function TileTomorrow({ pts, close }) {
  const implied = Math.round(close + pts);
  const signClass = pts > 0 ? 'text-bull' : pts < 0 ? 'text-bear' : 'text-neutral';
  return (
    <div className="bg-card rounded-xl p-6 shadow-sm">
      <div className="flex items-center justify-between">
        <h3 className="text-sm font-semibold">Tomorrow</h3>
        <span className="text-xs text-muted">Updated 05:30</span>
      </div>
      <div className="mt-3 flex items-baseline gap-4">
        <span className={`font-mono text-4xl ${signClass}`}>{pts > 0 ? `+${pts}` : `${pts}`} pts</span>
        <div className="text-xs text-muted">Implied: {implied}</div>
      </div>
      <Sparkline dataKey="close" data={history} range={[close - pts, close + pts]} />
    </div>
  );
}

10 — Chart & gauge choices (tools & justifications)

Sparklines & small charts: Recharts (lightweight, SVG, responsive).

Range bands: Recharts AreaChart with custom ReferenceLine for current price and filled Area for envelope.

Gauge / Dial for expansion prob: custom SVG ring with Framer Motion for sweep (fast, small).

Traffic-light tilt display: simple CSS grid with 3 cells, dynamic background color.

Pattern match progress bar: Tailwind width % with transition.

Modal detail: Plotly for larger interactive chart on demand (optional and lazy-loaded).

11 — Micro-interactions & motion

Tile entrance: fade + translateY 8px (duration 360ms).

Value update (numbers): subtle count-up animation (0.6s).

Gauge sweep: ease-out sweeping arc (0.6s).

Range changes: animated area morphing (animate area path with framer-motion or CSS transitions).

Keep motion restrained to avoid distraction.

12 — Accessibility & localization

All color-coded info must have text alternatives (e.g., show BULLISH label) for color-blind users.

Font sizes scalable with browser zoom; tiles responsive.

ARIA labels for interactive elements and tooltips.

Date/time shown with timezone (IST) and ISO timestamp in audit modal.
